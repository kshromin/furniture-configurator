# Структура проекта — Конфигуратор мебели

Коротко, для ориентира. Подробности — читаю сам по коду, когда нужно. **Обновляю при изменении структуры**, держу так же коротко.

Статический сайт без сборки: `index.html` + `css/style.css` + JS-модули в браузере.

Эта папка (`Config/config`) — только программа и git-репозиторий; рабочие материалы — уровнем
выше в `Config/` (Задания, Выгрузки, Бэкапы, Хостинг, Приложения, Ассортимент — см. `../CLAUDE.md`).

## Основные блоки

- **`js/core/`** — всё общее, не привязанное к типу мебели: 3D-сцена и камера, общее состояние проекта, UI-панель справа, перетаскивание элементов мышкой, размерные линии, заказ/пресеты, окно «Редактировать дверь» (doorEditor), печать сметы с предпросмотром (print), защита полей от автозаполнения браузера (autofillGuard).
- **`js/types/`** — по файлу на тип мебели (шкаф-купе, распашной, комод, стол и т.д.). Шкаф-купе — основной, самый проработанный тип.
- **Наполнение шкафа-купе** (полки/ящики/штанга/сетка/корзины/вешало) — самая объёмная часть, разложена на 4 файла по смыслу: «сколько места доступно» / «что где стоит и не пересекается» / «как это выглядит в 3D» / общие константы.
- **`data/materials.json`** — каталог материалов, цветов и цен.
- **Личный кабинет/логин** (auth/cabinet/admin) — отдельная, независимая фича, разрабатывается параллельно, к 3D-конфигуратору не относится.
