# Конфигуратор мебели

Параметрический 3D-конфигуратор мебели (Three.js) — статический сайт, без сборки. Открывать через `preview_start` (конфиг `static` в `.claude/launch.json`, порт 8723).

При старте работы в этой папке:
1. Прочитать [DEVLOG.md](DEVLOG.md) — журнал сессий (что менялось и почему), даже если не попросили явно.
2. Если что-то в git-состоянии выглядит непривычно (`git pull`/`push` ведут себя не так, как ожидалось) — см. [GIT-SYNC.md](GIT-SYNC.md).
3. [STRUCTURE.md](STRUCTURE.md) — карта файлов проекта (что где лежит). Держать в актуальном состоянии: любое изменение структуры (новый файл, разбивка/слияние/переименование модуля, смена ответственности между файлами) — сразу отражать там же, в рамках той же сессии, не откладывая.

Код синхронизируется между двумя компьютерами через GitHub (`git pull`/`push`), не через YandexDisk-копирование `.git`.

## Рабочий процесс (правила от пользователя, сессия 25)

- **Работаем по блокам**: типы изделий делаются ПО ОДНОМУ (сейчас — шкаф-купе). Кнопки
  не начатых типов заблокированы с подписью «в разработке» — белый список `READY_TYPES`
  в `js/core/tabs.js` (включить новый тип = добавить его id туда + убрать фильтр шаблонов
  в `js/core/presets.js`).
- **Коммитить** — локально по завершении фичи (без вопросов), это дёшево и спасает историю.
- **Пушить (`git push`) — ТОЛЬКО по команде «закончил»/«закончили»** — пуш выкладывает на
  сервер (GitHub Pages), пользователь проверяет локально до публикации. Не пушить после
  каждого коммита, как раньше.
- Экономить токены: не перечитывать большие файлы целиком, грепать точечно; STRUCTURE.md —
  карта, DEVLOG.md — контекст сессий.

**Триггерная фраза «закончили»/«закончил»**: полный вжик-вжик по завершению сессии, без
дополнительных вопросов:
1. Дописать в DEVLOG.md запись за сессию (что сделано и почему — как обычно).
2. Закоммитить все изменения (осмысленными коммитами, как в остальной истории).
3. Запушить на GitHub (`git push`) — единственный момент, когда пуш делается вообще.
